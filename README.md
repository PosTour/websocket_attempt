Второй класс: WebSocketConfig с WebSocketMessageBrokerConfigurer
<br>Этот класс включает поддержку WebSocket с использованием STOMP, что позволяет организовать полноценную систему сообщений на основе брокера (message broker). Это более высокий уровень абстракции, чем просто WebSocket, и подходит для реализации сценариев, связанных с публикацией/подпиской (publish/subscribe).

Основные моменты:
<br>Аннотация @EnableWebSocketMessageBroker: Включает поддержку WebSocket с использованием STOMP и брокера сообщений.
<br>Интерфейс WebSocketMessageBrokerConfigurer: Этот интерфейс предоставляет методы для настройки брокера сообщений и регистрация WebSocket STOMP точек (endpoints).

Метод configureMessageBroker():
<br>Включает простой брокер сообщений через enableSimpleBroker("/topic"), который позволяет клиентам подписываться на каналы сообщений с префиксом /topic.
<br>Устанавливает префикс /app для маршрутов, по которым будут отправляться сообщения от клиентов. Это означает, что сообщения, отправленные клиентом на адреса с этим префиксом, будут обработаны сервером.

Метод registerStompEndpoints():
<br>Регистрация STOMP WebSocket точки по URL /spring-boot-tutorial. Клиенты могут подключаться к этому адресу для установления WebSocket соединения.

Когда используется:
<br>Этот подход используется для построения систем обмена сообщениями на базе WebSocket с поддержкой STOMP протокола. STOMP значительно упрощает управление отправкой и получением сообщений между клиентом и сервером, особенно в многопользовательских системах (например, чаты, уведомления).
